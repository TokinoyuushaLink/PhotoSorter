# 视图更新机制

## @Observable 驱动的自动更新

Core 层的三个数据对象均使用 `@Observable` 宏：

```swift
@Observable final class PhotosStore { ... }
@Observable final class AlbumsStore { ... }
@Observable final class SortHistory { ... }
```

ContentView 中用 `@State` 持有这三个对象。SwiftUI 运行时在渲染时自动追踪访问了哪些属性，属性变化时只重绘依赖该属性的视图子树。

**关键属性及其触发的更新**：

| 属性变化 | 触发重绘的视图 |
|----------|---------------|
| `photosStore.assets` | PhotoGridView → NSCollectionView（差分更新） |
| `photosStore.selectedIDs` | PhotoCollectionView（选中状态高亮） |
| `photosStore.isLoading` | PhotoGridView（Loading / 空态 / 网格 三态切换） |
| `sortHistory.undoStack` | titlebarLayer（totalSortedAndDeleteCount → 模式按钮数字、✅ 按钮显隐） |
| `albumsStore.favoriteNodes` | AlbumStripCombined（收藏条芯片列表） |
| `albumsStore.recentNodes` | AlbumStripCombined（最近使用列表） |

## NSCollectionView 的差分更新

网格渲染的核心在 `PhotoCollectionView`（`NSViewControllerRepresentable`），它手动管理 NSCollectionView 数据，不走 SwiftUI diffing：

```
Coordinator.updateItems(new: [PhotoAsset])
    → 对比 old / new 列表
    → 若结构不同（sections 变化）→ collectionView.reloadData()（全量）
    → 否则 → collectionView.performBatchUpdates {
        deleteItems(at: toDelete)
        insertItems(at: toInsert)
        // 已有 item 就地更新（不移除，直接 setImage）
    }
```

这样归类一张照片时，只有该 cell 被删除，其余 cell 保持不动，不触发全量重载，保证动画流畅。

**选中状态同步**：
- `photosStore.selectedIDs` 变化 → `Coordinator` 通过 `onChange` 收到 → 调用 `applySelection()` → 直接操作 `collectionView.selectionIndexPaths`，不走 SwiftUI 重绘路径，避免 NSCollectionView 闪烁。

## 网格 → SinglePhotoView 的进场动画

进场流程需要精确的坐标变换和动画时序：

```
1. 用户双击 / Space
   focusedID = id
   focusedFrame = cell 在屏幕坐标的 frame（由 NSCollectionView 转换）
   enterSingleMode() 被调用

2. ContentView.enterSingleMode()
   singleModeAssets = 当前队列
   singleModeInitialIndex = index
   singleEnterTrigger += 1    ← 让 SinglePhotoView 知道是新的进场
   showingPreview = true       ← 挂载 SinglePhotoView（结构变化）
   isInSingleMode = true

3. SinglePhotoView.onAppear
   appeared = false
   sourceFrame = focusedFrame（缩略图位置）
   withAnimation(.spring) { appeared = true }
   → 图片从 sourceFrame 放大到全屏
   动画完成后：mediaVisible = true（显示视频/GIF 播放器）
```

## SinglePhotoView → 网格的退场动画

退场需要滚动网格、等待 cell 可见后再执行动画，时序较复杂：

```
1. 用户下滑 / Esc / Space
   dismissDragY > threshold → 触发退场

2. onBeforeDismiss(currentIndex)
   → ContentView 通知 PhotoCollectionView 滚动到该 index
   → collectionView.scrollToItems(at:)

3. onDismissBegin()
   → dismissBegun = true（ContentView 标记，用于屏蔽部分交互）

4. SinglePhotoView 执行退场动画
   → withAnimation(.spring) { appeared = false }
   → 图片缩回到 getGridFrame(currentIndex) 返回的 cell 位置

5. 动画完成 → onDismiss(currentIndex)
   → ContentView: pendingDismissWork 延迟执行
       showingPreview = false    ← 卸载 SinglePhotoView
       isInSingleMode = false
       dismissBegun = false
```

> `pendingDismissWork` 用 `DispatchWorkItem` 实现，可被下一次进场打断取消，避免"退场动画进行中又触发进场"时的状态冲突。

## NotificationCenter 驱动的跨层通信

SwiftUI 的数据绑定只能在视图树内向下传递，跨层或从 AppKit 向 SwiftUI 传递事件用 `NotificationCenter`：

| 通知名 | 发送方 | 接收方 | 用途 |
|--------|--------|--------|------|
| `.assetsDidSort` | PhotosStore.performAdd | ContentView | 显示顶部 hint、自动切换到已分类视图 |
| `.sortedViewShouldDismiss` | ContentView | ContentView | clearAll 后隐藏已分类视图 |
| `.sortedSelectionShouldClear` | ContentView | ContentView | 归类/撤销后清空选中状态 |
| `.refreshRequested` | AppDelegate 菜单 / Cmd+R | ContentView | 触发刷新（有已分类记录时弹确认） |
| `.autoHideStripToggled` | AppDelegate 菜单 | ContentView | 切换收藏条自动隐藏 |
| `.stripKeyDown` | KeyboardHandler | ContentView | 数字键长按时强制显示收藏条 |
| `.thumbnailFitToggled` | AppDelegate 菜单 | ContentView | 切换缩略图填充模式 |

## 键盘事件处理链

键盘事件从 NSEvent 全局监听器开始，经过多层过滤后到达业务逻辑：

```
NSEvent.addLocalMonitorForEvents(.keyDown)   ← KeyboardHandler（NSViewRepresentable）
    → 过滤：isInSingleMode 分支
        单图模式：
            数字键 1-0 → assignToAlbum（直接调用）
            Space → 退出单图模式
            ← → → 前后导航（发通知给 SinglePhotoView）
            Delete → moveToPendingDelete
            Cmd+Z / Cmd+Shift+Z → undo / redo
        网格模式：
            数字键 1-0 → assignToAlbum
            Space → 进入单图模式
            Delete → 删除/移动到待删除
            Cmd+A → 全选
            Cmd+\ → 切换侧边栏
```

## 滚动时的渐变更新

`PhotoGridView` 通过 `ScrollObserver`（NSView subclass）监听 NSScrollView 的 `NSScrollViewDidLiveScrollNotification`：

```
滚动事件 → ScrollObserver
    → contentOffset.y
    → topGradientOpacity = clamp((offsetY - topThreshold) / fadeRange, 0, 1)
    → bottomGradientOpacity = clamp((contentH - containerH - offsetY) / fadeRange, 0, 1)
    → 通过 @Binding 回写 ContentView 的 topGradientOpacity / bottomGradientOpacity
    → SwiftUI 更新渐变层 opacity
    → titlebarLayer 的 largeTitleOpacity 同步淡出（顶部渐变 0→0.5 段）
```

## 收藏条的悬停显示逻辑

`AlbumStripCombined` 在单图模式下默认隐藏，悬停底部区域显示：

```
TrackingAreaView（NSView）
    → mouseEntered → isHovered = true
    → mouseExited  → isHovered = false

shouldHide = isInSingleMode && autoHideInSingleMode && !isHovered && !forceShow
    → opacity: shouldHide ? 0 : 1（带动画）
    → allowsHitTesting: !shouldHide
```

`forceShow` 由数字键长按触发（`stripKeyDown` 通知），长按期间强制显示收藏条让用户看到快捷键提示。
