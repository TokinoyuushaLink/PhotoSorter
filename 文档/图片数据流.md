# 图片数据流

## 总览

```
Photos 框架 (PHPhotoLibrary)
    │
    ▼ loadUncategorized()
PhotosStore.assets [PhotoAsset]          ← 内存，按创建日期降序
    │
    ├─▶ PhotoGridView → PhotoCollectionView (NSCollectionView)
    │       │
    │       ▼ ThumbnailCache (Actor, LRU 400)
    │       ThumbnailView.NSView（缩略图渲染）
    │
    └─▶ SinglePhotoView
            │
            ├─▶ NSImage (assets[i].preview)     ← 全尺寸预览，按需加载
            ├─▶ AVPlayerView                    ← 视频
            └─▶ GIFPlayerView                   ← GIF 帧动画
```

## 阶段一：初始加载

**触发时机**：应用启动 → 权限授权完成 → `photosStore.loadUncategorized()`

```
[主线程] loadUncategorized()
    → 清空 assets / selectedIDs / phAssetCache / assetIndexByID
    → isLoading = true
    → 开启后台线程
        [后台线程]
        1. fetchAssetCollections(.album) → 枚举所有相册，收集已归类照片 ID 集合 idsInAlbums
        2. fetchAssets(所有照片，按 creationDate 降序)
        3. 过滤：!idsInAlbums.contains(id) → 只保留未归类照片
        4. 构建 [PhotoAsset]（不含图片数据，只有元数据）
        5. 构建 phAssetCache: [String: PHAsset]
    [回主线程]
        assets = result
        phAssetCache = cache
        assetIndexByID = 反查字典
        isLoading = false
        → loadPreviewsBatched()（委托给 PreviewLoader，批量触发预览图加载）
```

**PhotoAsset 结构**（纯元数据，无图片）：
```swift
struct PhotoAsset {
    let id: String           // PHAsset.localIdentifier
    let creationDate: Date?
    let pixelWidth: Int
    let pixelHeight: Int
    let mediaType: PHAssetMediaType
    var preview: NSImage?    // 延迟填充
    var isGIF: Bool
}
```

## 阶段二：缩略图加载（网格显示）

网格渲染走两条并行路径：

**路径 A：Cell 配置时同步查缓存（零延迟）**
```
PhotoCollectionView.collectionView(_:itemForRepresentedObjectAt:)
    → ThumbnailCache.cachedImage(for: id)   ← Actor，同步读，若命中立即显示
    → 若未命中，启动异步任务（路径 B）
```

**路径 B：异步加载并回填**
```
Task {
    → ThumbnailCache.image(for: id, phAsset:size:contentMode:)
        [内部]
        若缓存命中 → 返回
        否则 → PHImageManager.requestImage(...)（后台线程）
                  → 图片回来 → 存入 LRU cache（Actor 保护）
    → 回主线程：cell.setImage(image)
}
```

**ThumbnailCache（Actor）**：
- 容量：400 张
- LRU 淘汰：`accessOrder` 数组维护访问顺序，超限时移除最旧项
- 线程安全：Swift Actor 保证，所有访问自动串行化

**批量预加载（loadPreviewsBatched）**：
- `loadUncategorized` 完成后立即触发
- PhotosStore 把 `(id, PHAsset)` 列表传给 `PreviewLoader.loadBatched`
- PreviewLoader 按批次延迟异步加载，通过回调通知 PhotosStore 写入 `assets[idx].preview`

## 阶段三：单图预览加载

进入单图模式时，`SinglePhotoView` 按需加载当前及相邻帧：

```
enterSingleMode(at index)
    → singleModeAssets = assets（或选中子集）
    → singleModeInitialIndex = index
    → showingPreview = true（挂载 SinglePhotoView）
    → isInSingleMode = true

SinglePhotoView.body
    → currentItem = assets[currentIndex]
    → 若 currentItem.preview != nil → 直接显示 NSImage
    → 否则 → photosStore.loadPreviewIfNeeded(for: id)
                → PreviewLoader.loadIfNeeded(id:phAsset:completion:)
                [后台] PHImageManager 全尺寸加载
                [主线程回调] assets[idx].preview = image（触发重绘）
    → 媒体类型分支：
        .video  → AVPlayer 加载，AVPlayerView 显示
        .image + isGIF → GIFPlayerView（PHAssetResource 逐帧解码）
        .image  → ZoomableImageView（NSImageView，支持双击缩放）
```

**预加载相邻帧**：SinglePhotoView 在 `currentIndex` 变化时，异步预加载 `currentIndex±1` 的 preview，减少翻页白屏。

## 阶段四：归类操作（内存阶段）

```
用户操作（键盘数字键 / 点击收藏条 / 右侧面板）
    → ContentView.assignToAlbum(node)
        → photosStore.addIDs(ids, to: collection, albumNode: node, history: sortHistory)
            [PhotosStore.performAdd]
            → snapshots = 当前 assets 的拷贝（含 preview）
            → removeAssets(ids)     ← 立即从 assets 移除，网格消失
            → sortHistory.record(SortAction(.classify, node, snapshots))
            → 发出 .assetsDidSort 通知（触发顶部 hint）
        → albumsStore.recordRecent(node)   ← 更新最近使用列表
```

> 此时**没有任何 Photos 框架调用**，PHPhotoLibrary 未被修改。

## 阶段五：撤销（Undo）

```
Cmd+Z → sortHistory.popUndo()
    → SortAction(.classify, node, assets)
    → photosStore.restoreAssets(assets)
        → 按 creationDate 降序插回 assets 数组
        → 重建 phAssetCache / assetIndexByID
    → sortHistory.pushRedo(action)
    → 发出 .sortedViewShouldDismiss 或 .sortedSelectionShouldClear
```

## 阶段六：实际写入 Photos 框架

**触发时机 1：点击 ✅ 按钮（commitAll）**
```
sortHistory.groupedByAlbum
    → 按相册聚合所有 .classify 动作
    → 对每个相册：
        fetchPHAssets(for: assets)   ← PHAsset.fetchAssets，直查 Photos DB
        PHPhotoLibrary.performChanges {
            PHAssetCollectionChangeRequest(for: collection)?.addAssets(phAssets)
        } completion: { ok, err in
            DispatchQueue.main.async { done() }
        }
    → 若有 pendingDelete：
        PHPhotoLibrary.performChanges {
            PHAssetChangeRequest.deleteAssets(phAssets)
        }
    → 所有操作完成后：
        sortHistory.clearAll()
        showClassifiedView = false
        showTopHint("分类完成")
```

**触发时机 2：退出应用（commitAllAndQuit）**
```
AppDelegate.applicationShouldTerminate → .terminateLater
    → ContentView 接收到 quit 通知
    → 若有 pendingDelete → 先弹对话框
    → 用户确认 → commitAllAndQuit()
        → 同上，performChanges 全部完成后
        → NSApp.reply(toApplicationShouldTerminate: true)
```
