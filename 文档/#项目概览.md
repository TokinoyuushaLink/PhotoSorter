# 项目概览

PhotoSorterSUI 是一款 macOS 照片整理工具，让用户快速将"未归类"照片分配到 Photos 相册。核心交互是键盘/触控板驱动的批量归类，所有写入操作延迟到用户手动确认或退出时才真正提交到 Photos 框架。

## 技术栈

| 层面 | 技术 |
|------|------|
| UI 框架 | SwiftUI + AppKit 混合 |
| 数据框架 | Photos (PHPhotoLibrary) |
| 状态管理 | `@Observable`（Swift 5.9 宏） |
| 网格渲染 | NSCollectionView（性能关键路径） |
| 缓存 | Swift Actor（ThumbnailCache，LRU 400项） |
| 播放 | AVPlayer（视频）、手动帧解码（GIF） |
| 构建 | swiftc 直接编译，无 Xcode 工程文件 |
| 平台 | macOS 15.0+，arm64/x86_64 |

## 模块划分

```
App/       应用入口、生命周期、主容器视图
Core/      纯数据模型与业务逻辑（无 UI 依赖）
Grid/      照片网格（SwiftUI 壳 + NSCollectionView 实现）
Preview/   单图预览、手势导航、媒体播放
Panels/    右侧列浏览面板、底部收藏条
```

## 关键设计原则

**内存操作优先，写入延迟提交**
归类动作只修改内存（PhotosStore、SortHistory），UI 立即响应。实际的 `PHPhotoLibrary.performChanges` 只在两个时机触发：用户点击 ✅ 按钮（`commitAll`）或退出应用（`commitAllAndQuit`）。

**ZStack 层叠架构**
ContentView 用 8 个 ZStack 层管理所有 UI 层级，层与层之间通过 `@State` 变量和 `NotificationCenter` 协调，避免深层数据传递。

**SwiftUI + AppKit 边界**
性能敏感的路径（网格、收藏条芯片）用 AppKit 实现，包在 `NSViewRepresentable` / `NSViewControllerRepresentable` 壳里供 SwiftUI 使用。纯展示和状态逻辑用 SwiftUI。
