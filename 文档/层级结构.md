# 层级结构

## 文件树

```
PhotoSorterSUI/
├── App/
│   ├── main.swift                  # 应用入口，启动 NSApplication
│   ├── AppDelegate.swift           # 窗口配置、菜单、退出确认对话框
│   ├── AppViews.swift              # 工具视图：ResizeDivider、MovableTitleBarBackground、View.cursor
│   ├── ContentView.swift           # 主容器：State 声明、body、computed properties
│   ├── ContentView+Actions.swift   # 业务动作：assign、delete、commit、undo/redo
│   ├── ContentView+Layers.swift    # 8 个 ZStack 层的 @ViewBuilder 实现
│   └── ContentView+Navigation.swift# 单图模式进入/退出、队列构建、syncSingleMode
│
├── Core/
│   ├── Constants.swift             # 全局常量、Layout、Prefs、Anim、Notification 名、工具函数
│   ├── PhotosStore.swift           # 未分类照片列表状态、PHAsset 缓存、选中状态、归类写入
│   ├── PreviewLoader.swift         # 预览图加载（按需 + 批量），与 PhotosStore 解耦
│   ├── AlbumsStore.swift           # 相册树、收藏/最近列表、AlbumNode 定义
│   └── SortHistory.swift           # undo/redo 栈、pendingDelete 列表、groupedByAlbum 聚合
│
├── Grid/
│   ├── PhotoGridView.swift         # SwiftUI 包装层，处理空态/加载态，传参给 PhotoCollectionView
│   ├── PhotoCollectionView.swift   # NSCollectionView 实现：布局、差分更新、手势、滚动锚点
│   ├── ThumbnailView.swift         # 单个缩略图 NSView（图片 + 选中态 + 视频角标）
│   └── ThumbnailCache.swift        # Actor，LRU 缓存，最多 400 张缩略图
│
├── Preview/
│   ├── SinglePhotoView.swift       # 单图查看器：入场/退场动画、拖拽手势、媒体切换
│   ├── SwipeNavigationView.swift   # NSView 包装，捕获触控板滑动手势实现左右导航
│   ├── KeyboardViews.swift         # NSEvent 全局监听：数字键归类、Space、Delete、Cmd+Z 等
│   └── PlayerViews.swift           # AVPlayerView（视频）、GIFPlayerView（帧动画）
│
└── Panels/
    ├── AlbumStripView.swift        # 底部收藏条（NSCollectionView 芯片 + SwiftUI 行标签）
    └── ColumnBrowserView.swift     # 右侧列浏览面板（SwiftUI，递归展示相册树）
```

## ContentView 的 8 个 ZStack 层

ContentView.body 是一个 ZStack，从底到顶依次叠放：

| 层序 | 变量名 | 职责 | 所在文件 |
|------|--------|------|---------|
| 1 | `gridLayer` | PhotoGridView，网格主体 | ContentView+Layers |
| 2 | `backdropLayer` | 纯色背景（深色黑 / 浅色窗口色），随 previewOpacity 淡入 | ContentView+Layers |
| 3 | `singlePhotoView` | SinglePhotoView，单图预览；`isInSingleMode` 控制是否渲染 | ContentView+Navigation |
| 4 | `overlayGradientLayer` | 顶/底渐变遮罩 + 大标题 + 小标题，随滚动/单图模式淡入淡出 | ContentView+Layers |
| 5 | `albumStripLayer` | 底部收藏条（AlbumStripCombined） | ContentView+Layers |
| 6 | `rightPanelLayer` | 右侧列浏览面板（ColumnBrowserView）+ 拖拽分割线 | ContentView+Layers |
| 7 | `titlebarLayer` | 标题栏区域：模式切换控件、✅ 确认按钮、侧边栏开关 | ContentView+Layers |
| 8 | `undoRedoLayer` | 不可见层，监听 Undo/Redo/Delete 通知，转发给业务逻辑 | ContentView+Layers |

> 层 3（singlePhotoView）用 `isInSingleMode` 门控渲染，用 `previewOpacity` 驱动淡入淡出动画，两个变量分开是因为结构变化（渲染/不渲染）和动画进度需要独立控制。

## 状态变量分布

### ContentView（主协调层）
```swift
// 数据对象
@State photosStore: PhotosStore
@State albumsStore: AlbumsStore
@State sortHistory: SortHistory

// 视图模式
@State showClassifiedView: Bool     // 未分类 vs 已分类网格
@State isInSingleMode: Bool         // 是否进入单图预览
@State showingPreview: Bool         // 单图视图是否已挂载（gate）
@State previewOpacity: CGFloat      // 单图淡入淡出进度

// 单图导航
@State singleModeAssets: [PhotoAsset]   // 当前预览队列（全部 or 多选子集）
@State singleModeInitialIndex: Int
@State singleModeCurrentIndex: Int

// 布局
@State rightWidth: CGFloat          // 右侧面板宽度
@State sidebarVisible: Bool

// 渐变
@State topGradientOpacity: CGFloat
@State bottomGradientOpacity: CGFloat
```

### PhotosStore（数据层）
```swift
var assets: [PhotoAsset]            // 未分类照片列表
var selectedIDs: Set<String>        // 当前选中
var isLoading: Bool
var authStatus: PHAuthorizationStatus
private var phAssetCache: [String: PHAsset]   // id → PHAsset，供写入用
private var assetIndexByID: [String: Int]     // id → assets 下标，O(1) 查找
private let previewLoader: PreviewLoader      // 预览图加载（解耦注入）
```

### PreviewLoader（预览图加载）
从 PhotosStore 解耦出的独立类，通过回调写回数据，不持有 Store 引用。
```swift
func loadIfNeeded(id:phAsset:completion:)   // 按需加载单张（进入单图时）
func loadBatched(items:completion:)         // 批量预加载（loadUncategorized 完成后）
```

### SortHistory（操作层）
```swift
private var undoStack: [SortAction]
private var redoStack: [SortAction]
private var pendingDeleteAssets: [PhotoAsset]  // 待永久删除
```
