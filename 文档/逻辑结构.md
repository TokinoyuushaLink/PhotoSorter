# 逻辑结构

## 核心数据模型关系

```
AlbumNode (AlbumsStore)
    ├── id: String                  ← PHAssetCollection.localIdentifier
    ├── title: String
    ├── kind: .album(PHAssetCollection) | .folder(PHCollectionList)
    └── children: [AlbumNode]?      ← nil=叶节点(album), []=空文件夹

SortAction (SortHistory)
    ├── kind: SortActionKind
    │     ├── .classify(albumNode)          ← 正常归类
    │     ├── .returnToUncategorized(node)  ← 从已分类移回
    │     ├── .moveToPendingDelete(node?)   ← 移入待删除
    │     └── .restoreFromPendingDelete     ← 从待删除恢复
    └── assets: [PhotoAsset]        ← 操作时的快照（含 preview）

SortHistory
    ├── undoStack: [SortAction]     ← 可撤销的操作序列
    ├── redoStack: [SortAction]     ← 可重做的操作序列
    └── pendingDeleteAssets: [PhotoAsset]  ← 待永久删除（退出时执行）
```

## SortHistory.groupedByAlbum 聚合逻辑

`commitAll` 和"已分类"视图的数据来源：

```
遍历 undoStack 中所有 .classify 动作
    → 按 albumNode.id 分组（保持首次出现顺序）
    → 同一相册的 assets 合并，按 creationDate 降序排列
    → 返回 [(albumNode, [PhotoAsset])]
```

注意：撤销操作会从 undoStack 移除对应 action，自动从聚合结果中消失。

## 归类操作的完整生命周期

```
[触发]  键盘数字键 / 收藏条点击 / 右侧面板点击
    ↓
[路由]  ContentView.assignToAlbum(node)
    → 单图模式：singleModeAssets[currentIndex] 的 id
    → 多选模式：selectedIDs（已分类视图）
    → 普通模式：focusedID ?? selectedIDs
    ↓
[过滤]  ContentView.assignIDs(ids, to: node)
    → 检查是否在 pendingDelete 中 → 若是，先从 pending 移除（restoreFromPendingDelete 路径）
    → 检查是否已在其他相册中（sortHistory.groupedByAlbum）→ 若是，先从旧相册移除
    ↓
[执行]  photosStore.addIDs(ids, to: collection, albumNode: node, history: sortHistory)
    → PhotosStore.performAdd:
        1. 从 assets 数组移除（UI 立即消失）
        2. sortHistory.record(.classify)
        3. 发 .assetsDidSort 通知
    ↓
[反馈]  albumsStore.recordRecent(node)
        顶部 hint 显示相册名
```

## 撤销/重做逻辑

```
Undo (Cmd+Z):
    action = sortHistory.popUndo()
    switch action.kind:
    case .classify(node):
        photosStore.restoreAssets(action.assets)   ← 插回 assets
        sortHistory.pushRedo(SortAction(.returnToUncategorized, assets))
    case .returnToUncategorized(node):
        photosStore.removeAssets(ids)
        sortHistory.record(.classify(node, assets))
        sortHistory.pushRedo(...)
    case .moveToPendingDelete(node?):
        sortHistory.removeFromPendingDelete(ids)
        若 node != nil → 重新 classify 到原相册
        否则 → restoreAssets 回 uncategorized
    case .restoreFromPendingDelete:
        photosStore.removeAssets(ids)
        sortHistory.addPendingDelete(assets)

Redo (Cmd+Shift+Z):
    action = sortHistory.popRedo()
    → 重新执行对应操作（和正常归类路径相同）
```

## 已分类视图（showClassifiedView）

切换到"已分类"标签时，PhotoGridView 接收的数据来源从 `photosStore.assets` 变为 `classifiedSections`：

```
classifiedSections: [SectionData]
    ← sortHistory.groupedByAlbum.map { (node, assets) in
        SectionData(header: node.title, assets: assets)
    }
    + 若有 pendingDelete → 追加一个"待删除"分组
```

已分类视图下的删除操作（Delete键 / 工具栏删除按钮）：
```
returnSelectedToUncategorized(ids)
    → sortHistory.groupedByAlbum 中找到对应 assets
    → sortHistory.record(.returnToUncategorized)
    → sortHistory.removeAssetsFromHistory(ids)
    → photosStore.restoreAssets(assets)
```

## 退出流程

```
用户 Cmd+Q
    → AppDelegate.applicationShouldTerminate → .terminateLater
    → 发 NSApplication.willTerminateNotification
    → ContentView 接收：
        若 sortHistory.hasPendingDeletes:
            → showQuitConfirm = true（弹"先删除待删除"对话框）
        else if sortHistory.totalSortedCount > 0:
            → showQuitUncategorized = true（弹"确认归类后退出"对话框）
        else:
            → NSApp.reply(toApplicationShouldTerminate: true)

用户点"确认归类后退出":
    → commitAllAndQuit()
        → 对每个相册执行 performChanges
        → 全部完成 → NSApp.reply(.terminateNow)

用户点"不归类直接退出":
    → sortHistory.clearAll()
    → NSApp.reply(.terminateNow)
```

## 相册树构建

`AlbumsStore.reload()` 在启动时和刷新时调用：

```
[后台线程] buildTree()
    → fetchAssetCollections(.album, .any) → 所有用户相册
    → fetchCollectionLists(.albumFolder)  → 所有文件夹
    → 构建 AlbumNode 树（递归 buildFolderNode）
    → 过滤掉系统智能相册（subtype 前缀判断）
[主线程]
    roots = tree
    favoriteIDs → favoriteNodes（保持 UserDefaults 中的顺序）
    recentAlbumIDs → recentNodes（最多 20 个）
```

**收藏排序**：favoriteIDs 存在 UserDefaults，`reorderFavorites(from:to:)` 直接操作 favoriteIDs 数组并持久化，UI 通过 `favoriteNodes` 的 `@Observable` 变化自动更新。

## 触控板滑动手势（SwipeNavigationView）

```
NSPanGestureRecognizer（attachedto NSView）
    .began:
        过滤鼠标位置：排除右侧面板区域、底部收藏条区域
        检测滑动方向（水平 or 垂直）
    .changed:
        水平：navOffset = translation.x → 图片跟手平移
        垂直（向下）：dismissDragY = translation.y → 退场拖拽
    .ended:
        水平速度/位移 > 阈值 → 切换到下一张（withAnimation）
        垂直位移 > 阈值 → 触发退场
        否则 → 弹回（spring 动画）
```

手势识别器通过 `shouldRecognizeSimultaneously` 与 ScrollView 共存：垂直方向优先让 ScrollView 处理，水平方向优先让 PanGesture 处理。
